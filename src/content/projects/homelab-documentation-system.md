## What it is

A lightweight documentation setup for homelab services, scripts, machine notes,
and recurring maintenance tasks.

## Why I made it

Undocumented systems become archaeology. I want future fixes to start from known
context instead of memory.

## Stack / tools

- Linux
- Docker
- Markdown
- GitHub
- Static site notes

## What worked

- Keeping service notes close to the repo makes them easier to update.
- Markdown is fast enough that I actually use it.
- Small templates reduce friction when documenting a fix.

## What broke

- Naming matters more than expected. Bad filenames make notes hard to find.
- Too much structure slows down quick notes.

## What I learned

The best documentation system is the one that catches notes while the problem is
still fresh. I am optimizing for low friction first, polish second.
